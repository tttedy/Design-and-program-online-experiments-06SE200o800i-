## On-Class Practical Exercise Workflow


### 1. Get the Latest Updates
Open VS Code

Go to Source Control

Click the Sync button to pull the latest changes

#### 1.1  sync/merge your fork with new changes from original repo

In VS code terminal, type the following:

git remote add upstream https://github.com/J2K101000101/Design-and-program-online-experiments-06SE200o800i-.git

git fetch upstream

git checkout main

git merge upstream/main

git push origin main

#### 1.2  If you encounter conflict merge, please either see[ git_merge_conflict.html ](https://j2k101000101.github.io/Design-and-program-online-experiments-06SE200o800i-/git_merge_conflict.html) or Session1/STARTER/create_your_folder_md/If you ever have a Git divergence issue...

### 2. Never Edit the STARTER Files Directly

Under each session folder, you'll find a STARTER folder containing the exercise materials. 

🙅‍♀️ IMPORTANT: Do NOT edit files directly in the STARTER folder.

### 3. Create Your Personal Workspace

Copy the entire STARTER folder

Paste it into the same session folder

Rename it with your name (e.g., marry_J, john_doe, your_name)

Work on your exercise only inside your personal folder


### 4. Save and Submit Your Work
After completing your exercise or making any changes:

Commit your changes with a descriptive message 

Push/Sync Changes to GitHub
